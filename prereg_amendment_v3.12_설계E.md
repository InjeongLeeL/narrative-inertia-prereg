## 1. 근거 — 대리라벨 지위 재확인 (무변경)

v3.11 §0.1: LLM 라벨 = **비인증 대리변수(surrogate)**. DSL(Egami et al.)은 대리 오류가 **임의의 비무작위 편향이어도 일치추정·유효 신뢰구간**을 보장한다. 따라서:
- 대리라벨은 **인증급(gold급) 건별 정밀 판정을 요구하지 않는다.** 대리 품질 저하는 **효율(DSL 분산 절감 크기)에만** 영향하며 추정의 **편향·타당성에는 영향이 없다.**
- 최저 보장선: gold-only 층화 HT/Hájek 추정(v3.11 §4.1-1①, 1,475건+원장 π만, 대리 불사용)이 **대리 품질과 무관한 바닥 추정**을 제공. 사전 정밀도 표(§2.6)의 gold-only 기준선이 유지된다.

## 2. 정정·경량화 조항 (효력 있는 규범)

### 정정 1 — 대리라벨 판정 프로토콜 경량화 (v3.11 §3.1 판정 심도 supersede)
- **배치 분류 허용:** 응답당 일괄 판정. 배치 크기·구성은 정정 4-2의 결정적 규칙을 따른다.
- **결정트리 약식 적용:** 핵심 3단(STEP1 referent → STEP2 in/out(종착 술어 요지) → STEP3 role)만 적용. 판례·재정 세목의 건별 전문 대조는 불요. **측정 정의(5범주·in/out 매핑·코드북 v1.4 준거)는 동일 — 심도만 상이.**
- **rationale = 한 구절 태그로 충분**(예: "dso: 특허 데이터원", "subst: 국가 혁신시스템"). 전문 서술 불요.

### 정정 2 — 정밀 판정 규율의 적용 범위 명확화 (무변경 존치)
v3.11의 인증급 정밀 판정(삭제 테스트·종착 술어·판례 전문·§0.3 코드북 v1.4+판례)은 **gold 코딩(§2.7: 인간 2코더 독립 + PI 중재)에만** 적용된다. 대리라벨엔 정정 1의 약식 프로토콜을 적용한다.

### 정정 3 — 생성 모델 변경: Opus → Sonnet 계열 (v3.11 §1.3 고정 대상 supersede)
- 전 49청크 대리라벨 = **Sonnet 계열 단일 모델**로 고정. **"전 청크 model_string 동일" 규칙 자체(v3.11 §1.3)는 불변** — 고정 대상만 변경. 상이 발생 시 즉시 중단 규율 유지.
- **정확 model_string은 사전 하드코딩하지 않는다(★r2):** 파일럿(§4) 첫 유효 응답의 실측 model_string을 **decision log에 기재·확정**하고, 그 값이 이후 전 청크 동일성 규율의 기준값이 된다. (명명 체계 불일치로 인한 불필요 수정안 발행 방지 — 사전등록 강도 불감소: "단일 값 고정" 원칙은 동결 시점에 이미 구속.)
- 사유: Opus 수백 세션 실행 불능.

### 정정 4 — 신규 동결 산출물 3종 + 배치 유효성 세칙 (★r2 신설 — 차단급 해소)

1. `prompt_template_v2.0_batch.md` — 배치용 system 프롬프트 + 약식 3단 결정트리 + 배치 출력 형식 명세. **V 유래 앵커 14건의 승계 여부(전부/일부/미승계)를 본문에 공시.**

**4-2. 배치 구성·유효성·시도 규칙(명문):**
- **배치 구성의 결정성:** 청크 내 dyad_id **canonical 오름차순 · 고정 크기 50** 분할(마지막 잔여 배치 <50 허용). 배치 구성에 난수·재량 없음(동일 입력→동일 배치).
- **각 배치 완료 즉시 저장·progress·부분 봉인**(v3.11 §3.1 운영 골격의 배치 단위 적용 — 배치=크래시 안전 체크포인트, 청크(500) 단위 병합·봉인은 유지).
- **dyad_id echo 필수:** 응답의 모든 판정에 dyad_id 명기. **매칭은 echo된 dyad_id 기준 — 위치(순서) 기준 매칭 금지.**
- **dyad 단위 유효성 판정(시도 계수 단위 = dyad, v3.11 불변 유지):** 배치 응답 내 각 dyad에 대해 —
  - 파싱 유효성 4조건(v3.11 §1.3: 허용 범주 외·category–status 정합 위반·필수 필드 누락·confidence∉[0,1]) 위반 = 해당 dyad invalid.
  - **echo 누락**(요청 dyad가 응답에 없음) = 해당 dyad invalid. **중복 echo**(동일 dyad_id 2회 이상) = 해당 dyad invalid(응답 간 선택 금지 원칙의 배치 확장). **미요청 id echo** = 해당 판정 무시·기록만.
  - invalid가 아닌 dyad는 **채택**(배치 전체 재시도 금지 — 유효분 보존).
- **재시도:** 실패 dyad만 수집 → canonical 오름차순·고정 크기 50으로 **재배치**하여 후속 시도. dyad당 총 3회(최초1+재시도2, v3.11 불변) 초과 = §3.3 규칙(dso/out/imputed=1)로 병합 시 확정.
- **raw 보존:** 전 배치 원응답 raw + batch_id↔dyad_id 매핑 로그 보존(v3.11 raw 보존 원칙의 배치 확장).

**4-3. 연속성 게이트(§1.6)의 지위 재정의(★r2):** 기 통과된 연속성 게이트는 **v1.6 계보 증빙으로 완료·존치**되며, v2.0 배치 템플릿에는 적용되지 않는다. v2.0의 측정 정의 동일성은 별도 payload 대조가 아니라 **코드북 v1.4 준거 명시(정정 1)와 템플릿 동결·공시**로 담보한다. (v1↔v2 payload 동일성 주장 불성립 — 구조 상이 명기.) **v2.0 측정 타당성의 유일한 실측 점검은 파일럿 §4의 3항(V-dev sanity)이며, 이는 실행·보고 필수 단계다(비게이트 — 판정·임계값 없음, 허용 반응은 §4의 5항 그대로).**

**★불변(무변경) 명시:** 출력 스키마(§3.2) · 파싱 유효성 4조건(§1.3) · dyad당 3회 시도(§1.3) · NA=병합 확정 imputed=1(§3.3) · render_population_v1.py(a9f6c66f…) 존치 · 배분/추출/시드/commitment/원장/3단 게이트/MC(§1.4~§2.5) · 모집단 24,149(§2.1) · gold 코딩·중재·신뢰도(§2.7~2.8) · 분석 사양(§4) · 49청크 단위·progress·부분 봉인(§3.1 운영 골격). 전부 그대로.
